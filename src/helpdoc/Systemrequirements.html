System requirements

Getting Started ››

This software runs anywhere Python 3.10 runs — Windows, Linux and macOS. The system hardware requirement depends entirely on the size of the rasters you're modeling; there is no R dependency at all. Because the geospatial stack (GDAL/rasterio/geopandas) relies on binary libraries, Conda or Mamba is the recommended way to install dependencies.

Option 1: Conda/Mamba (recommended)

Step 1. Install Miniforge/Mamba (or Anaconda/Miniconda) as per its instructions.

Step 2. From the repository root, create the environment from the bundled environment.yml:

       mamba env create -f environment.yml

Step 3. Activate it:

       mamba activate openldm

+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

Option 2: Pip only (no Conda/Mamba available)

You must manually ensure GDAL's C++ headers are installed on your system first, then:

       pip install -r requirements.txt

Use one option or the other, never both in the same environment — layering requirements.txt on top of a Conda env can overwrite the Conda-managed pyqtwebengine build with a mismatched pip build and break the GUI's Help tab.
